# VibeCoding — Evidence · Flow · Automation

이 문서는 **SSoT(단 하나의 기준)** 를 중심으로,
아이디어 → 스펙 → AI 초안 → 반복(Iteration) → MVP → 확장 → 배포 흐름을
**증거(Evidence)** 와 함께 운영하기 위한 공개용 문서입니다.

- 로컬(Obsidian)에서 Dataview로 운영 상태를 자동 집계합니다.
- 원격(GitHub)에서는 MkDocs로 정적 문서를 자동 빌드/배포합니다.
- SSoT의 핵심 요약은 README에 자동 반영되어, 항상 최신의 “한 장 요약”을 유지합니다.

핵심 원칙: **기준은 SSoT, 변경은 기록, 배포는 자동화**.
